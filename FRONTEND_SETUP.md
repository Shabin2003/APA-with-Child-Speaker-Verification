# Frontend Setup Guide

## Prerequisites

- Node.js 18+ or Bun
- A running FastAPI backend (see `backend/` and `FIREBASE_SETUP.md`)
- Firebase project with Auth enabled (see `FIREBASE_SETUP.md`)

---

## 1. Environment Variables

Create a `.env` file in the **project root** (same level as `package.json`):

```env
# Firebase Web App config — get from Firebase Console → Project Settings → Your apps
VITE_FIREBASE_API_KEY=AIzaSy...
VITE_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your-project
VITE_FIREBASE_STORAGE_BUCKET=your-project.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=123456789
VITE_FIREBASE_APP_ID=1:123456789:web:abc123

# FastAPI backend URL — no trailing slash
VITE_API_URL=http://localhost:8000
```

**`VITE_API_URL` options:**

| Environment | Value |
|---|---|
| Local development | `http://localhost:8000` |
| Colab + Cloudflare Tunnel | `https://xxxx-xxxx.trycloudflare.com` |
| Colab + ngrok | `https://abc123.ngrok.io` |
| Production server | Your deployed API URL |

> ⚠️ The URL must have **no trailing slash**. The frontend strips trailing slashes internally but it's better to be consistent.

## 5. User Roles

The app supports two roles set at signup:

| Role | Access |
|---|---|
| **Student** | Pronunciation assessment, speaker enrollment, personal profile |
| **Teacher** | All student access + admin dashboard showing all students and their assessment history |

Role is stored in MongoDB (`users` collection) and checked server-side on every admin request.

## 6. Features

### Speaker Enrollment
1. Go to **Profile → Enroll Voice**
2. Record **3 or more** voice samples (audio or video mode)
3. Each sample is processed by ECAPA-TDNN to extract a 192-dim speaker embedding
4. Embeddings are stored in MongoDB linked to your Firebase UID
5. Once 3+ samples are enrolled, `enrolled: true` appears on your profile

> You must complete enrollment before speaker verification activates during pronunciation assessment.

### Pronunciation Assessment
1. Go to **Pronunciation Assessment**
2. Choose difficulty: **Easy** (ages 7–10) · **Medium** (ages 11–14) · **Hard** (ages 15–18)
3. A sentence is generated by **Groq Llama 3.1** tailored to your level
4. Click 🔊 to hear the correct pronunciation via **Kokoro TTS** (falls back to browser speech synthesis if unavailable)
5. Record yourself reading the sentence (audio or video)
6. Click **Assess Pronunciation** — the backend runs ASR and speaker verification **in parallel**
7. Results show:
   - **Accuracy %** — `(1 - WER) × 100`
   - **Word Error Rate (WER)** — word-level edit distance
   - **Character Error Rate (CER)** — character-level edit distance
   - **Your transcript** vs the reference sentence
   - **Identity verification** badge — ACCEPT / REJECT / Not enrolled

### Teacher Dashboard
- Available only to users with `role = teacher`
- Shows all students with their last 20 assessment sessions
- Displays per-student average accuracy and total session count

## 7. API Endpoints Used by the Frontend

All endpoints except `/generate-text` and `/sentences/pool` require a Firebase Bearer token in the `Authorization` header. The frontend handles this automatically via `authHeader()` in `src/lib/api.ts`.

| Endpoint | Method | Auth | Used for |
|---|---|---|---|
| `POST /pronunciation/analyze-and-verify` | POST | ✅ | Main assessment (ASR + verification) |
| `POST /pronunciation/analyze` | POST | ✅ | Standalone pronunciation scoring |
| `POST /asr/transcribe` | POST | ✅ | Game transcription |
| `POST /speaker/enroll` | POST | ✅ | Voice enrollment |
| `DELETE /speaker/reset` | DELETE | ✅ | Clear and re-enroll |
| `POST /speaker/verify` | POST | ✅ | Standalone verification |
| `GET /generate-text?level=Medium` | GET | ❌ | Fetch sentence + TTS audio URL |
| `GET /sentences/pool` | GET | ❌ | Offline sentence fallback |
| `POST /user/profile` | POST | ✅ | Create / update profile |
| `GET /user/profile` | GET | ✅ | Fetch own profile |
| `GET /admin/students` | GET | ✅ Teacher | All students + assessments |

**Form-data shape for enrollment and assessment:**

```
POST /speaker/enroll
  speaker_id: string   (must equal your Firebase UID)
  age: integer
  file: audio/video blob

POST /pronunciation/analyze-and-verify
  text: string         (the reference sentence)
  age: integer
  file: audio/video blob
```

All audio/video blobs are decoded server-side — WebM, MP4, OGG, and WAV are all supported.

## 8. Troubleshooting

### "Failed to fetch" / CORS errors
- Check `VITE_API_URL` in `.env` is correct and has no trailing slash
- Confirm the FastAPI server is running and reachable
- If using Cloudflare Tunnel or ngrok, check the URL hasn't changed since the server restarted
- Open browser DevTools → Network tab to see the actual request URL

### Audio/video not recording
- Allow microphone (and camera for video mode) when the browser prompts
- HTTPS is required on some browsers for MediaRecorder — Cloudflare Tunnel and ngrok both provide HTTPS automatically
- Try **Audio Only** mode if camera permissions are blocked
- If issues persist on Safari, update to iOS 14.5+

### TTS button (🔊) not playing
- The button always appears — if Kokoro TTS is unavailable on the server it falls back to browser `SpeechSynthesis`
- Check browser console for audio load errors
- Ensure `VITE_API_URL` doesn't have a double-slash before `/audio/`

### Speaker verification always REJECT
- Ensure you have enrolled **3 or more** voice samples
- If you changed the backend speaker model, existing embeddings are incompatible — call `DELETE /speaker/reset` and re-enroll
- Enroll using the same recording conditions (microphone, noise level) as during assessment

### Colab backend disconnects
- Colab sessions time out after ~90 minutes of inactivity
- Re-run the backend cell and update `VITE_API_URL` with the new tunnel URL
- The frontend will show "API not responding" errors when the backend is down
